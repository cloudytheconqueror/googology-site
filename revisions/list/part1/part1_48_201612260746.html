Negative Loader's number
-D5(99)

The negative of the largest named computable number, whose definition can be turned into (for example) a Turing machine instruction or a C code. Loader's number is the output of loader.c, a C program code written by Ralph Loader at the Bignum Bakeoff contest. The goal of the contest was to write a C code that produces the largest possible result on a theoretical infinite-memory machine. The C code should contain no more than 512 non-blank symbols.

Loader's program diagonalizes over the calculus of constructions, a powerful theory.

Negative one
-1

This is a special negative number, and the negative of the most important number in mathematics.

Some example of -1's use in mathematics are that any nonzero number to the power of -1 is its reciprocal (x^(-1) = 1/x), the square of the imaginary unit i^2 is equal to -1, and the famous identity e^(iπ) = -1, which can be used to find out what taking a number to the power of a complex number means.

-1 plays a vital role in googology, as it's used to define the predecessor of any integer. It is used in the definition of the predecessor function:

P(n) = n-1

As well as almost all other googological functions, such as the hyper operator:

Rule 1. a {1} b = a + b
Rule 2. a {n} 1 = a (if n>1)
Rule 3. a {n} b = a {n-1} (a {n} (b-1)) (if n>1)

The subtraction of 1, which is equivalent to the addition of -1, is needed to force the evaluation of any expression back to the base case (Rules 1 and 2 in this case), so that the notation eventually terminates. Looking in this way -1 has some actual significance in googology.

Sbiis Saibian suggests this as a boundary between "small" negative numbers and "large" negative numbers. Any negative number less than -1 is "large" and any negative number more than -1 is "small" (i.e. close to 0).

Another fact: According to this blog post, infinity is less than zero. In fact, it's equal to -1.

Double logarithm of two
log(log(2)) = -0.521390227654...

This number, although negative, has an important role in googology. Say, you want to convert 2^3^4^5 into a power tower of 10's with a "smallish" number at the top. For an accurate conversion we use logarithms:

2^3^4^5
= (10^log(2))^3^4^5 — 10^(log(x)) = x by definition
= 10^(log(2)*3^4^5)
= 10^(10^log(log(2))*(10^log(3))^4^5)
= 10^10^(log(log(2))+log(3)*4^5)
~ 10^10^(-0.521390+0.477121255*1024)
~ 10^10^(-0.521390+488.572165)
= 10^10^488.050775

This shows that if you want to calculate the topmost exponent with the precision of at least one digit after decimal point, you need to take log(log(2)) into account. This could be a big deal, considering that an increase of the topmost exponent by 0.5 results in the number (approximately) being cubed:

10^10^(n+0.5) = 10^(10^n*10^0.5) ~ 10^(10^n*3.1622) = (10^10^n)^3.1622

Imagine a bucket containing 10^10^n grains of sand. Imagine there are 10^10^n of such buckets in a storage room, and imagine 10^10^n such storage rooms. That's about what's needed to have 10^10^(n+0.5) sand grains. That's the difference between 10^10^n and 10^10^(n+0.5). Although in practical calculations in googology we ignore these differences as those are insignificant, sometimes when detailed calculation is needed, it is necessary to consider small numbers like this.

Zero
0

We reached the important number 0, the only real number that's neither negative nor positive.

Sbiis Saibian considers zero as the smallest quantity possible, and negative numbers are considered "less than zero" rather than "smaller than zero".

It is the additive identity — x + 0 = 0 + x = x for any number x. For other operations it leads to some degenerate cases: x*0 = 0, x^0 = 1, x^^0 = 1, x^^^0 = 1 and so forth.

Zero is equal to the logarithm of 1 and the double logarithm of 10: log(1) = log(log(10)) = 0. For logarithms with other bases, the double logarithm of the base equals 0: log_b(log_b(b)) = log_b(1) = 0.

In order for algorithm to terminate, a minimum value is necessary for every argument, which is typically chosen as 1 or 0. Therefore zero has some importance in googology as a minimum value, such as in the Ackermann function, and the minimum value for an entry in a separator in Bowers' array notation. Zero is also the smallest ordinal, and f_0(x) in the fast-growing hierarchy is defined as x+1.

The name "Non-positive number range" was a bit inaccurate, as zero, a non-positive number, is not in it. But it's the only non-positive number that won't be in that range.

Okojo-ermine number
Oe ~ 1/{54,55 (1) 2}

This is a very small number, defined by the Japanese googologist Aeton (Aetonal on GWiki) in 2013.

"Okojo" is a Japanese word with two meanings: "ermine", an okojo in winter fur, and "stoat", an "okojo" in summer fur.

Define the following:
  • f(n) = x where x satisfies (10↑↑n)^(10↑↑n) = (10↑)^(n+2) x — here (10↑)^a b is a power tower of a 10's with b at the top (or Ea#b in Hyper-E)
  • f(1,1,□) = f(54,□)
  • f(1,□,n) = f(1/(f(1,□,n-1)),□) where 1/(f(1,□,n-1)) needs to be rounded off to an integer. It is unclear which way it rounds off to.
  • f(■,m,1,□) = f(■,m-1,54,□)
  • f(■,m,□,n) = f(■,m-1,1/(f(■,m,□,n-1)),□) — also needs to be rounded off
Where:
  • □ is zero or more 1's separated by commas
  • ■ is zero or more integers larger than or equal to 1, separated by commas
  • m > 1 and n > 1
Using the f function, the okojo numbers are defined as:
  • Oe(n) = f(1,1,...,1) with n+1 1's, Os(n) = 1/Oe(n)
  • Okojo-ermine number (Oe) = Oe(54)
  • Okojo-stoat number (Os) = Os(54) = 1/Oe (roundoff not necessary)
The number 54 often appears in the definition because o, ko, jo can be read as 0, 5 and 4, respectively (although we're stretching a bit on 4, which is read as yon).

Common logarithm of two
log(2) = 0.301029995664...

A small number that raising 10 to the power of this number results in 2. This number has an important role in googology. Since 10^log(2) = 2, it can be used to change the base of a power tower from 2 to 10. For example, when we calculate 2^2^30:

2^2^30
= 2^1,073,741,824
= (10^log(2))^1,073,741,824
= 10^(log(2)*1,073,741,824)

... and putting log(2)*1,073,741,824 in a calculator gives 323,228,496.622955. Therefore...

2^2^30 ~ 10^323,228,496.622955

This shows that this number (and other logarithmic numbers) is useful for changing the power's base from 2 to 10 to determine where powers of numbers other than 10 fall between the powers of 10. The negative number log(log(2)) is another number with a similar role in googology.

The logarithms of prime numbers are also useful for computing logarithms of composite numbers by factorizing them and by taking the sum of the logarithms of the factors. For example we can compute the logarithm of 32 using the logarithm of 2:

log(32) = log(2^5) = log(2)*5 = 0.301029996*5 = 1.50514998

If we knew logarithms of more prime numbers (such as log(3) and log(5)) we could compute the logarithm of even more numbers.

This number satisfies the equality 10^log(3) = 3. Using log(2) and log(3), we can estimate the logarithm of some positive integers without a calculator. For example:

After all these small numbers, we finally reached what's arguably the most important number in mathematics.

One is the smallest positive integer, and is the only positive integer that's neither prime nor composite. It is the multiplicative identity: x*1 = 1*x = x for all real number x.

Adding 1 could be considered the "zeroth operation" — the successor function S(n) satisfies S(n) = n+1, and it's considered one of the fundamental functions of googology.

Four is an integer that comes between 3 and 5, or S(3) using the successor function. Unlike smaller positive integers, there aren't many special words about four, except "quarter". Four can still be easily recognized, so it's considered a palpable number.

4 is the first composite number, the second square number and the first perfect power (apart from 0 and 1), the second tetrahedral number (between 1 and 10), the third Lucas number (between 3 and 7), the first non-Fibonacci number, and the 3rd Motzkin number (between 2 and 9). It's the output of BB(2) (busy beaver function), making it a busy beaver number (between 1 and 6). It's also the output of Ξ(4) (xi function) and H(2) (hyperfactorial). The four-color theorem states that four is the minimum number of colors need to color any two-dimensional map on a plane such that adjacent countries have different colors. 4 is also the maximum number of squares needed to sum up to any positive integer.

The prefixes for four are tetra- (ex. tetrahedron) and quad- (ex. quadruple). Both are commonly used.

In googology, 4 is significant as the result of 2+2, 2*2, 2^2, 2^^2, 2^^^2 and all higher operators. There are many things grouped in fours, since a set of four gives a feel of completeness to it. Square root of 62
sqrt(62) = 7.874 007874 011811 019685 034449...

The square root of 62 has a remarkable digit pattern, as shown above. 7874 is repeated, 11811 is 7874 times 1.5, and 19685 is 7874 times 2.5. Amazingly, this is not a coincidence, it arises from the fact that 999,998 = 62*127^2.

First, we note that the reciprocal of 999,998 and its square root has the following digit pattern:

1/999,998 = 1.000002000004000008000016000032...
sqrt(1/999,998) = 0.001000001000001500002500004375...

Since 7874 = 999,998/62, multiplying both sides of the latter equation by 7874 gives:

sqrt(62) = 7874*sqrt(1/999,998) = 7874 * 0.001000001000001500002500004375... = 7.874007874011811019685034448...

A similar pattern can be seen in the square root of 2: sqrt(2) = 1.4 14 21 35 62... In this case, it results from the fact that 2*7^2 = 98.

For more about this phenomenon, see Robert Munafo's entry on sqrt(62) and his separate article "Fractions with Special Digit Sequences".

Nine / garthree

9 is the 3rd square (between 4 and 16), the 4th Motzkin number (between 4 and 21), and the smallest odd composite number. It can be named garthree using the gar- prefix. Catalan's conjecture, which was proven in 2002, states that 8 = 2^3 and 9 = 3^2 are the only pair of perfect powers (aside from 1st powers) that differ by 1.

9 is the maximum number of cubes needed to sum up any positive integer. However, 23 = 2^3*2+1^3*7 and 239 = 4^3*2+3^3*4+1^3*3 are the only integers that actually need nine cubes, and beyond that only eight cubes are needed. It is proven that at some point only seven cubes are necessary, but it's not proven about six or fewer cubes, although it's been proven that at least four cubes are needed to sum up any positive integer beyond a certain point.

It is equal to q(4) (Laver table), the largest known value of the sequence (0, 2, 3, 5, 9). For more about Laver tables, see A(9,A(8,A(8,255))). 9 is the largest single-digit number—beyond this point we need the help of place-value notation. Because 9 is one less than the base of our numeral system, 10, 100, 1000 and so on are all congruent to 1 mod 9. From this a simple method of divisibility test by 9, called "casting out nines". Take the sum of the digits of the number. The original number is divisible by 9 if and only if the sum is. For example, 653,589 is divisible by 9 because 6+5+3+5+8+9 = 36 and 36 is a multiple of 9.

The prefixes for nine are nona-/nov- (Latin, ex. November) and ennea- (Greek, ex. enneagon). Sometimes the nona- prefix is used in place of ennea- even when Greek prefixes are used on other numbers.

Booga-e by Nayuta Ito's proposal (old)
e^(e^(e-2)-e+3) = 10.304951874639...

Nayuta Ito of Googology Wiki made a proposal to generalize the definition of up-arrows to any positive real arguments, as well as defining fractional up-arrows. In this proposal we can define things like e^^^^pi or 4.6{2.5}3.8. The proposal has an old and a new version. The rules of the old version are given below:
  1. b>1 and n>1: a{n}b = a{n-1}(a{n}(b-1))
  2. 0≤b≤1 and n>1: a{n}b = a^b
  3. 0≤b≤1 and 0≤n≤1: a{n}b = a^b^n*b^(1-n)
A new version of the proposal is created to fix the issue that 2{n}2 is not always equal to 4 (for example, 2{0.5}2 = 2^2^0.5*2^(1-0.5) = 3.769083...). The first and second rules are the same, but the third rule is changed to:
  1. 0≤b≤1 and 0≤n≤1: a{n}b = a^(1+n(b-1))*b^(1-n)
Booga-e is defined with Sbiis Saibian's booga- prefix, which is defined as booga-x = x{x-2}x. For example, booga-3 = 3^3 = 27 and booga-4 = 4^^4 ~ 10^(8*10^153). Eleven
11

11 is the 5th prime number (between 7 and 13) and the 5th Lucas number (between 7 and 18). 11 is the only palindromic prime with an even number of digits, because palindromes with an even number of digits are divisible by 11.

11 is the smallest repunit prime. Repunit primes are primes whose decimal digits are all 1. If all decimal digits of a prime number (larger than 10) are all the same, then it should be 1 because if it was another number, then the original number would be a multiple of that digit. The next repunit prime is R19 = 1,111,111,111,111,111,111.

11 is equal to n(2) in the block sequence theorem, which is the length of the longest string x_1, x_2, ... of the letters 1 and 2 such that no block of letters xi, ..., x2i is a substring of any later block xj, ..., x2j (j>i). The longest such strings that start with 1 are "12221111111" and "12221111112". Strings starting with 2 can be created by changing 1 to 2 and vice-versa.

The square of 11 is 121, which spells out the second row of Pascal's triangle (counting the row with the single 1 as the 0th row). This continues with the next two powers: 1331, 14641. The pattern breaks at the next power of 11, which is 161,051. Powers of 1.1 work similarly, except with a decimal point after the first 1. Powers of 101, 1001 etc. also work similarly, and goes further than the powers of 11.

This also applies to other bases: in base n, powers of n+1 are represented in base n as 11, 121, 1331, ... this pattern continues until we reach a row in Pascal's triangle whose largest entry is larger than or equal to n.

There are three ways to test for divisibility by 11. In each test, the original number is a multiple of 11 if and only if the result is a multiple of 11 (in two-digit numbers, have the same two digit).
  1. Alternating addition and subtraction, starting from the right. Example: 1-2+9-5+8 = 11, so 85,921 is a multiple of 11.
  2. Adding digits in groups of two, again starting from the right. The process is repeated until the result is smaller than 100 (this works because 11 is a multiple of 99). Example: 8+59+21 = 88, so 85,921 is a multiple of 11.
  3. Repeatedly subtract the last digit from the rest of the digits. Example: 8592-1 = 8591, 859-1 = 858, 85-8 = 77, 7-7 = 0, so 85,921 is a multiple of 11. What's useful about this way is that it can be used to determine the quotient as well: write the subtracted digits in reverse order. In 85,921's case, the subtracted digits are 1, 1, 8, 7, so the quotient is 7811.
The name "eleven" doesn't match with the pattern for thirteen to nineteen—the etymology for "eleven" is "one left".

The prefixes for eleven are undeci- (Latin) and endeka- (Greek). At this point the prefixes are created by combining smaller parts, and their usage is becoming quite rare.

"Up to eleven" is a common idiom for topping the previous maximum, since things often go from a scale from zero/one to ten. It is usually credited to have originated from the film This Is Spinal Tap.

Ramsey's theorem is a problem related to Graham's problem, the problem that gave rise to Graham's number. It involves coloring the lines of a complete graph red and blue (a complete graph Kn is a set of n vertices with every pair of points connected with a line). Ramsey's theorem states that for any x or y, for a sufficiently large n all two-colorings of Kn will have a red Kx and/or a blue Ky as a subgraph.

Define the function R(x,y) as the smallest integer n such that all two-colorings of Kn will have a red Kx and/or a blue Ky as a subgraph. The numbers of the form R(x,y) are known as Ramsey numbers. In particular, R(4,4) = 18. Note that R(y,x) = R(x,y) since you can just swap the colors.

Aside from the trivial cases R(1,y) = 1 and R(2,y) = y, only nine non-trivial Ramsey numbers R(x,y) (where x≤y) are known as of May 2016, and these are:
  • R(3,3) = 6
  • R(3,4) = 9
  • R(3,5) = 14
  • R(3,6) = 18
  • R(3,7) = 23
  • R(3,8) = 28
  • R(3,9) = 36
  • R(4,4) = 18
  • R(4,5) = 25
In other cases the exact value is not known, but some bounds are known:
  • 43 ≤ R(5,5) ≤ 49
  • 102 ≤ R(6,6) ≤ 165
  • 205 ≤ R(7,7) ≤ 540
  • 282 ≤ R(8,8) ≤ 1870
  • 565 ≤ R(9,9) ≤ 6588
  • 798 ≤ R(10,10) ≤ 23,556
The Wikipedia article on Ramsey's theorem lists more bounds.

A Menger sponge is a 3D fractal, which is constructed as follows:

51 is equal to Worm(2) by a function defined using Beklemishev's worms, a construction described by Lev D. Beklemishev. Define a worm as a sequence of non-negative integers [W0, W1, ..., Wn], and then solve it using what Beklemishev calls "the Worm battle". On turn m (starting from m=1), the worm W is converted to next(W,m), which is defined as follows:
  • If Wn = 0, next(W,m) = [W0, W1, ..., Wn-1]. In other words, the last entry is chopped off.
  • Otherwise:
    • Find the rightmost entry of W smaller than Wn, and call that Wk.
    • If such Wk exists, define the good part as g = [W0, ..., Wk], and the bad part as b = [Wk+1, ..., Wn-1, Wn-1] (the last entry is decremented).
    • If no such Wk exists, define g as an empty list and b = [W0, ..., Wn-1, Wn-1].
    • Then next(W,m) = g + b + ... + b with m+1 copies of b, where + means sequence concatenation, for example [3,2,3]+[4,9,6] = [3,2,3,4,9,6].
Beklemishev proved that the worm is eventually reduced to [] no matter what worm W is picked, and that it can't be proven in Peano arithmetic. Now we can define Worm(n) as the number of steps it takes to reduce the worm [n].

Solution of x^x = 10^100

When you raise this number to its own power, the result is equal to a googol. This number can be represented exactly as e^W(ln(10^100)) = e^W(100*(ln(2)+ln(5))), where W is the Lambert W-function. The Lambert W-function is the inverse function of f(W) = W*e^W.

In general, the solution of x^x = k is e^W(ln(k)). This can be proved in this way: for any w, W(w)*e^W(w) = w by definition, so letting w = ln(k) gives W(ln(k))*e^W(ln(k)) = ln(k). Therefore k = e^(W(ln(k))*e^W(ln(k))) = (e^W(ln(k)))^(e^W(ln(k))). Letting x = e^W(ln(k)) gives k = x^x, completing the proof.

61 is expressible as A(3,3) in the Ackermann function. There are multiple variations of the Ackermann function, but this one is the most well-known. The function is defined as follows:
  • a = 0: A(0,b) = b+1
  • b = 0: A(a,0) = A(a-1,1)
  • Otherwise: A(a,b) = A(a-1,A(a,b-1))
Using the definition directly, A(3,3)'s computation begins with:

A(3,3)
= A(2,A(3,2))
= A(2,A(2,A(3,1)))
= A(2,A(2,A(2,A(3,0))))
= A(2,A(2,A(2,A(2,1))))
= A(2,A(2,A(2,A(1,A(2,0)))))
= A(2,A(2,A(2,A(1,A(1,1)))))
= A(2,A(2,A(2,A(1,A(0,A(1,0))))))
= A(2,A(2,A(2,A(1,A(0,A(0,1))))))
= A(2,A(2,A(2,A(1,A(0,2)))))
= A(2,A(2,A(2,A(1,3))))
= A(2,A(2,A(2,A(0,A(1,2)))))
= A(2,A(2,A(2,A(0,A(0,A(1,1))))))
= A(2,A(2,A(2,A(0,A(0,A(0,A(1,0)))))))
= A(2,A(2,A(2,A(0,A(0,A(0,A(0,1)))))))
= A(2,A(2,A(2,A(0,A(0,A(0,2))))))

= A(2,A(2,A(2,A(0,4))))

= A(2,A(2,A(1,A(1,A(1,3)))))
= A(2,A(2,A(1,A(1,A(1,A(1,2))))))

... and it goes on and on, until it reaches the result 61. As you can see, using the definition directly is an inefficient way of calculating this function. There are actually some simple formulas that makes computing the Ackermann function easier, given below:
  • A(0,b) = b+1
  • A(1,b) = b+2
  • A(2,b) = 2b+3
  • A(3,b) = 2^(b+3)-3
  • A(4,b) = 2^^(b+3)-3
  • A(5,b) = 2^^^(b+3)-3
  • A(a,b) = 2^^...^^(b+3)-3 (with a-2 arrows)

The Fibonacci sequence is defined as F1 = 1, F2 = 1, and Fn = Fn-1 + Fn-2 for n≥3. In other words, the first two terms are 1, and each term after that is the sum of the two before it. Starting the sequence with 0 and 1 yields the same sequence, so 0 could be called the "zeroth Fibonacci number".

The Fibonacci sequence arises from this famous question by Leonardo Pisano (Leonardo of Pisa, also known as Fibonacci): "Say that the rabbits produce a pair of offspring every month, starting two months after they are born. If we start with a pair of young rabbits, how many pairs will be there after a year?"

On the first month there is the original pair. On the second month a new pair is born, so there are two pairs. On the third month the original pair produces a new pair, and the second pair grow up, so there are three pairs. On the fourth month two pairs are born, bringing the total to five pairs, and so on. After 12 months passed, there are a total of F14 = 377 pairs.

The Fibonacci sequence grows exponentially, with the ratio between two consecutive entries approaching the golden ratio phi = 1.618033... This property was found in the 18th century, and the occurrences of Fibonacci numbers in nature (flower petals, for example) was noticed in the 19th century.

196 is the smallest Lychrel number in base 10. To define Lychrel numbers, consider the following process:
  1. Take any number n
  2. Add n's reverse to n
  3. Repeat step 2 until the result is a palindrome (reads the same forwards and backwards, e.g. 59,395)

292 is a number that appears in one of the first few entries of π's continued fraction expansion. A continued fraction is a way to express a number like so:

a + 1/(b + 1/(c + 1/(d + 1/(e + ...))))

The above continued fraction can be written more simply as a; b, c, d, e...

It has been proven that the length of the continued fraction of a number is finite if and only if the number is rational, and there are exactly two continued fraction representations if the number is rational, or a unique representation if it's irrational.

Some irrationals have nice continued fractions, such as e: 2; 1, 1, 4, 1, 1, 6, 1, 1, 8, 1, 1, ... while some don't, such as π: 3; 7, 15, 1, 292, 1, 1, 1, 2, 1, 3, 1, 14, ... (OEIS A001203)

What's notable about pi's continued expression is that you can cut off everything from 292 and beyond:

3 + 1/(7 + 1/(15 + 1/(1 + 1/(292 + ...))))

to make:

3 + 1/(7 + 1/(15 + 1/1)) = 3 + 1/(7 + 1/16) = 3 + 1/(113/16) = 3 + 16/113 = 355/113 = 3.141592920354...

which is a famous approximation to π that matches up to the first six decimal digits. The reason why this approximation is so accurate is because the discarded terms 1/(292 + ...) is quite small, and doesn't affect the result by a big amount. Additionally, cutting the terms from 15 and beyond creates the approximation 22/7.

There are also generalized continued fractions, where the numerators need not be 1. It can be used to express π with a nice pattern, such as 4/(1 + 12/(2 + 32/(2 + 52/(2 + ...)))) or 2 + 2/(1 + 1*2/(1 + 2*3/(1 + 3*4/(1 + ...)))).

381 is equal to g(6) using the weak Goodstein function. The function is defined using weak Goodstein sequences, which are defined below:

Suppose we want to create a weak Goodstein sequence starting with n. First, represent n in base 2, for example 36 = 2^5+2^2. Then the next term is calculated by bumping the base to 3 and subtracting one. In the case of 36, we get 3^5+3^2 - 1 = 251. For example, the weak Goodstein sequence for 36 goes as follows:
  • g0(36) = 36 = 25 + 22
  • g1(36) = 35 + 32 - 1 = 251 = 35 + 2*3 + 2
  • g2(36) = 45 + 2*4 + 2 - 1 = 1033 = 45 + 2*4 + 1
  • g3(36) = 55 + 2*5 + 1 - 1 = 3135 = 55 + 2*5
  • g4(36) = 65 + 2*6 - 1 = 7787 = 65 + 6 + 5
and so on.

728 is a Smith number, a composite number where the sum of its digits is equal to the sum of all the digits appearing in its prime factorization. In 728's case, 728 = 2*2*2*7*13 and 2+2+2+7+1+3 = 17, which is equal to 7+2+8.

The first (not-so) few Smith numbers are 4, 22, 27, 58, 85, 94, 121, 166, 202, 265, 274, 319, 346, 355, 378, 382, 391, 438, 454, 483, 517, 526, 535, 562, 576, 588, 627, 634, 636, 645, 648, 654, 663, 666, 690, 706, 728, 729, 762, 778, ... (OEIS A006753) Note that 728 and 729 is the first occurrence of consecutive integers. Such pair is called a Smith brother pair.

Smith numbers were named by Albert Wilansky, who discovered the property in 4,937,775, the phone number (493-7775) of his brother-in-law Harold Smith.
