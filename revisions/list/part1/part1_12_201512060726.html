Negative infinity
-∞

Negative infinity is a quantity that is less than all real numbers. It is not very useful to googology as it's not on the real number line, but it shows up when calculating the logarithm of zero (in any base), and taking the double logarithm of one.

Negative BIG FOOT
-FOOT10(10^100)

This is the negative of what that is currently considered the largest named number, BIG FOOT. It is the result of applying the FOOT function to googol 10 times, where FOOT(n) is defined as the largest number definable in first-order oodle theory (FOOT for short) using x or less symbols. LittlePeng9 of Googology Wiki came up with the definition, and Sbiis Saibian suggested the name.

Negative Rayo's number
-Rayo(10^100)

The negative of the former largest named number, until BIG FOOT was defined. Rayo's number is defined as the smallest number larger than any number expressable in first-order set theory with a googol or less symbols. Despite the simple definition, there's no way to compute the number.

Negative Loader's number
-D5(99)

The negative of the largest named computable number, whose definition can be turned into (for example) a Turing machine instruction or a C code. Loader's number is the output of loader.c, a C program code written by Ralph Loader at the Bignum Bakeoff contest. The goal of the contest was to write a C code that produces the largest possible result on a theoretical infinite-memory machine. The C code should contain no more than 512 non-blank symbols.

Loader's program diagonalizes over the calculus of constructions, a powerful theory.

Negative godsgodgulus
-E100#{100}#100

The negative of a number around the limit of Saibian's Extended Cascading-E Notation (xE^), which is currently one of the largest numbers Saibian defined using the Extensible-E system (ExE). It is not the largest number Saibian named—see BIG FOOT for that.

Negative tethrathoth
-E100#^^#100

This the negative of a number around the limit of Saibian's Cascading-E Notation (E^), the 3rd part of the Extensible-E system. This number is much smaller than the above, but it's still large enough to be around the limit of Peano arithmetic.

Negative gongulus
-{10,10 (100) 2}

This number is the negative of a number that is much smaller than the absolute values of the previous entries, but is still really big. It is the result of solving a 100-dimensional 10×10×...×10 hypercube of 10's in extended array notation.

Negative gongulus was mentioned in Jonathan Bowers' article "Why Does God Exist?", to make a point that every number has "trueness". This results in negative gongulus the least number mentioned on Bowers' entire site.

Negative iteral
-{10,10,10,10,10,10,10,10,10,10}

The negative of a googolism that's at the breaking point of linear array notation. It's still huge, right, but small enough that once you learn the (relatively simple among Bowers' array notations) ruleset you can understand how this is computed.

Negative Graham's number
-G(64)

The negative of Graham's number, a very well-known googolism. Because if a number is well-known, somebody might mention the negative of it.

Negative giggol
-10^^100

The negative of a number that's so amazingly small that it can be written using iterated exponentiation as 10^10^...^10 (with 100 10's). We are surely getting closer to zero!

Negative googolplex
-10^10^100

The negative of one of two so-well-known numbers that the name "googology" is based on it. It is defined as 1 followed by googol zeroes, where googol is 1 followed by 100 zeroes.

Negative googol
-10^100

The negative of the smaller cousin of googolplex, which is 1 followed by 100 zeroes. It's so small that the universe has more Planck volumes than this number!

Negative million
-1,000,000

We have gotten to the negative of a number that is so small that it has everyday meaning. Imagine having this as a debt (assuming you are using a denomination that has similar value to the US dollar)—that would suck.

Negative forty
-40

This number has a notable property on temperature: Celsius and Fahrenheit temperature scales give equal values at -40 (-40 degrees C = -40 degrees F). For every increase by 1 of the Celsius degree, the Fahrenheit degree increases by 1.8.

Double logarithm of one point zero zero zero zero zero one
log(log(1.000001)) = -6.36221590585...

As x approaches 1 from above, the double logarithm of x approaches negative infinity. But the approach happens slowly, that if you use x = 1.000001, the result is only about -6. If you enter x = 1.000...001, the number you get will be roughly equal to the number of digits after the decimal point of x. As you can see this method isn't very effective for generating large negative numbers.

Negative one
-1

This is a special negative number, and the negative of the most important number in mathematics.

Some example of -1's use in mathematics are that any nonzero number to the power of -1 is its reciprocal (x^(-1) = 1/x), the square of the imaginary unit i^2 is equal to -1, and the famous identity e^(iπ) = -1, which can be used to find out what taking a number to the power of a complex number means.

-1 plays a vital role in googology, as it's used to define the predecessor of any integer. It is used in the definiton of the predecessor function:

P(n) = n-1

As well as almost all other googological functions, such as the hyper operator:

Rule 1. a {1} b = a + b
Rule 2. a {n} 1 = a (if n>1)
Rule 3. a {n} b = a {n-1} (a {n} (b-1)) (if n>1)

The subtraction of 1, which is equivalent to the addition of -1, is needed to force the evaluation of any expression back to the base case (Rules 1 and 2 in this case), so that the notation eventually terminates. Looking in this way -1 has some actual significance in googology.

Sbiis Saibian suggests this as a boundary between "small" negative numbers and "large" negative numbers. Any negative number less than -1 is "large" and any negative number more than -1 is "small" (i.e. close to 0).

Another fact: According to this blog post, infinity is smaller than zero. In fact, it's equal to -1.

Double logarithm of two
log(log(2)) = -0.521390227654...

This number, although negative, has an important role in googology. Say, you want to convert 2^3^4^5 into a power tower of 10's with a "smallish" number at the top. For an accurate conversion we use logarithms:

2^3^4^5
= (10^log(2))^3^4^5 — 10^(log(x)) = x by definition
= 10^(log(2)*3^4^5)
= 10^(10^log(log(2))*(10^log(3))^4^5)
= 10^10^(log(log(2))+log(3)*4^5)
~ 10^10^(-0.521390+0.477121255*1024)
~ 10^10^(-0.521390+488.572165)
= 10^10^488.050775

This shows that if you want to calculate the topmost exponent with the precision of at least one digit after decimal point, you need to take log(log(2)) into account. This could be a big deal, considering that an increase of the topmost exponent by 0.5 results in the number (approximately) being cubed:

10^10^(n+0.5) = 10^(10^n*10^0.5) ~ 10^(10^n*3.1622) = (10^10^n)^3.1622

Imagine a bucket containing 10^10^n grains of sand. Imagine there are 10^10^n of such buckets in a storage room, and imagine 10^10^n such storage rooms. That's about what's needed to have 10^10^(n+0.5) sand grains. That's the difference between 10^10^n and 10^10^(n+0.5). Although in practical calculations in googology we ignore these differences as those are insignificant, sometimes when detailed calculation is needed, it is necessary to consider small numbers like this.

Zero
0

We reached the important number 0, the only real number that's neither negative nor positive.

Sbiis Saibian thinks zero as the smallest quantity possible, and negative numbers are considered "less than zero" rather than "smaller than zero".

It is the additive identity — x + 0 = 0 + x = x for any number x. For other operations it leads to some degenerate cases: x*0 = 0, x^0 = 1, x^^0 = 1, x^^^0 = 1 and so forth.

Zero is equal to the logarithm of 1 and the double logarithm of 10: log(1) = log(log(10)) = 0. For logarithms with other bases, the double logarithm of the base equals 0: log_b(log_b(b)) = log_b(1) = 0.

In ordrer for algorithm to terminate, a minimum value is necessary for every argument, which is typically chosen as 1 or 0. Therefore zero has some improtance in googology as a minimum value, such as in the Ackermann function, and the minimum value for an entry in a separator in Bowers' array notation. Zero is also the smallest ordinal, and f_0(x) in the fast-growing hierarchy is defined as x+1.

The name "Non-positive number range" was a bit inaccurate, as zero, a non-positive number, is not in it. But it's the only non-positive number that won't be in that range.

Okojo-ermine number
Oe ~ 1/{54,55 (1) 2}

This is a very small number, defined by the Japanese googologist Aeton (Aetonal on GWiki) in 2013.

"Okojo" is a Japanese word with two meanings: "ermine", an okojo in winter fur, and "stoat", an "okojo" in summer fur.

Define the following:
  • f(n) = x where x satisfies (10↑↑n)^(10↑↑n) = (10↑)^(n+2) x — here (10↑)^a b is a power tower of a 10's with b at the top (or Ea#b in Hyper-E)
  • f(1,1,□) = f(54,□)
  • f(1,□,n) = f(1/(f(1,□,n-1)),□) where 1/(f(1,□,n-1)) needs to be rounded off to an integer. It is unclear which way it rounds off to.
  • f(■,m,1,□) = f(■,m-1,54,□)
  • f(■,m,□,n) = f(■,m-1,1/(f(■,m,□,n-1)),□) — also needs to be rounded off
Where:
  • □ is zero or more 1's separated by commas
  • ■ is zero or more integers larger than or equal to 1, separated by commas
  • m > 1 and n > 1
Using the f function, the okojo numbers are defined as:
  • Oe(n) = f(1,1,...,1) with n+1 1's, Os(n) = 1/Oe(n)
  • Okojo-ermine number (Oe) = Oe(54)
  • Okojo-stoat number (Os) = Os(54) = 1/Oe (roundoff not necessary)
The number 54 often appears in the definition because o, ko, jo can be read as 0, 5 and 4, respectively (although we're stretching a bit on 4, which is read as yon).
